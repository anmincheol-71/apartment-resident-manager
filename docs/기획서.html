1. 프로젝트 개요

본 프로젝트는 아파트 관리사무소 담당자가 입주민 정보 관리, 보안 이상징후 모니터링, 입주민 공지사항 발송을 하나의 프로그램에서 처리할 수 있도록 개발한 Qt 기반 데스크톱 관리 시스템이다. 데이터 저장은 별도 서버 없이 로컬 SQLite DB를 사용하며, Qt6의 Multimedia 모듈을 활용해 CCTV 영상 재생 기능도 내장하였다.

화면은 상단 네비게이션 바와 QStackedWidget 구조로 구성되어 있으며, 입주민 관리 · 보안 알림 내역 · 공지사항 세 페이지를 탭처럼 전환하는 방식을 택했다. 하단 상태바에는 총 등록 세대 수와 미확인 보안 알림 건수가 실시간으로 표시된다.

2. 요구사항 및 기능 정의

입주민 정보 관리 (CRUD)

입주민의 동·호·이름·연락처·입주일·메모를 등록·수정·삭제할 수 있으며, 세대당 차량을 최대 3대까지 함께 관리한다. 입주민을 삭제하면 연결된 차량 정보도 자동으로 삭제되도록 SQLite 외래 키 제약(ON DELETE CASCADE)을 적용하였다.

검색은 이름, 연락처, 차량번호 세 가지 기준을 지원한다. 차량번호 검색의 경우 resident 테이블과 car 테이블이 분리되어 있어 서브쿼리를 활용하여 해당 차량을 소유한 입주민을 조회한다. 목록 테이블에서 차량번호 열은 GROUP_CONCAT 집계 함수로 다중 차량을 한 셀에 표시하며, 이를 위해 QAbstractProxyModel을 서브클래싱하여 동적으로 열을 추가하는 방식을 구현하였다.

CCTV 보안 알림

이상징후가 감지되면 화면 우측 하단에 경고 팝업이 항상 최상단에 고정되어 표시된다. 팝업에서는 발생 위치·유형·시각 확인, CCTV 영상 재생, 확인 완료 처리, 공지사항 등록을 바로 수행할 수 있다. 프로그램 시작 5초 후에는 지하주차장 침입 감지 시나리오가, 12초 후에는 단지 내 의심인물 배회 시나리오가 자동으로 실행되어 실제 운용 상황을 시연할 수 있도록 하였다.

공지사항

공지는 일반 · 보안경고 · 주의인물 세 가지 유형으로 작성할 수 있으며, 수신 대상은 전체 세대, 특정 동, 특정 세대 단위로 지정할 수 있다. 동·호 목록은 입주민 DB에서 실시간으로 불러오므로 별도 입력 없이 콤보박스에서 선택하는 방식이다. 보안 알림 팝업의 '주의 공지 등록' 버튼을 누르면 알림 유형을 분석해 공지 유형과 제목이 자동으로 채워진 상태로 작성 창이 열려 빠른 등록이 가능하다.

3. UI / UX 설계

전체 화면은 상단 네비게이션 바와 콘텐츠 영역으로 나뉜다. 네비게이션 버튼을 클릭하면 QStackedWidget이 해당 페이지로 전환되며, 각 페이지는 독립적인 레이아웃을 가진다.

입주민 관리 페이지에서 행을 더블클릭하면 세대 상세정보 팝업이 열리며, 등록 차량 목록을 확인하고 수정 버튼으로 바로 편집할 수 있다. 보안 알림 내역 페이지에서는 이력 테이블의 행을 더블클릭하면 해당 알림의 CCTV 영상이 별도 창에서 재생된다.

보안 알림 팝업

이상징후 발생 시 아래와 같은 팝업이 화면 우측 하단에 항상 최상단으로 고정 표시된다. Qt::Tool · FramelessWindowHint · WindowStaysOnTopHint 플래그를 조합하여 다른 창에 가리지 않도록 구현하였다.

4. 데이터 구조

데이터는 SQLite 파일 하나에 네 개의 테이블로 관리된다. 입주민(resident)과 차량(car)은 1:N 관계로 연결되며, 보안 알림(security_alert)과 공지사항(notice)은 독립적으로 저장된다.
